# 05. DSSM —— 工业界最主流的双塔召回模型

> 对应教材：fun-rec 第 2.3.2 节
> 论文：Huang et al., 2013（微软搜索）
> 工业地位：抖音、淘宝、YouTube 召回主力模型

## 🎯 这一节的里程碑

| 之前的模型 | DSSM |
|---|---|
| 只用 ID embedding | **任意特征 → DNN → 向量** |
| 用户/物品建模耦合 | **用户塔 + 物品塔，完全独立** |
| 没有归一化 | **L2 归一化 + 温度系数** |
| 学术玩具 | **真实工业架构** |

## 核心架构

```
用户特征 → 用户塔(DNN) → L2归一化 → user_emb
物品特征 → 物品塔(DNN) → L2归一化 → item_emb
                              ↓
                    score = (u·v) / τ
                              ↓
                          softmax loss
```

## 工程亮点

- **离线**：所有物品过塔 → 存进 FAISS/HNSW 等 ANN 索引
- **在线**：用户来时实时算用户向量 → ANN 查 Top-K（< 10ms）
- **训练-检索一致性**：归一化让"点积 = 欧氏距离"，线上检索准确

## 关键设计

| 设计 | 作用 |
|---|---|
| DNN 替代单一 embedding | 学习非线性特征交互 |
| L2 归一化 | 训练点积 = 线上欧式距离 |
| 温度系数 τ | 调节 softmax 锐度 |
| 用户/物品塔独立 | 离线预算 + 在线检索 |
| In-batch negatives | 每个 batch 内互相做负样本，效率高 |
